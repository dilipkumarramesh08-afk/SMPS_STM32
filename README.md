# STM32F103C6T6 PSFB Closed-Loop Firmware

Target MCU is STM32F103C6T6 low-density STM32F103, 32 KB Flash and 10 KB SRAM. Firmware uses CMSIS/direct register access only. No HAL, Arduino, runtime menu, command parser, dynamic allocation, or printf is used.

## Operating Mode

This firmware is **true PSFB closed-loop output-voltage control only**.

Not present:
- no open-loop mode
- no open-loop estimated-voltage mode
- no basic balanced PWM mode
- no runtime mode selection
- no serial menu

Output voltage is set only at the top of [src/main.c](src/main.c):

```c
#define TARGET_VOUT_MV 24000U
```

Allowed range is `24000U` to `28000U`. If the value is outside this range, PWM is never enabled and `FAULT_INVALID_TARGET` is latched.

## Hardware

| Function | Pin | Peripheral |
|---|---:|---|
| Q1 left high-side command | PA8 | TIM1_CH1 |
| Q2 left low-side command | PB13 | TIM1_CH1N |
| Q3 right high-side command | PA9 | TIM1_CH2 |
| Q4 right low-side command | PB14 | TIM1_CH2N |
| Future current trip | PB12 | TIM1_BKIN reserved |
| Output feedback | PA0 | ADC1_IN0 |
| SWD | PA13/PA14 | untouched |

Gate-driver inputs must have external pull-down resistors, for example 10 kOhm. Small series resistors from MCU to gate-driver inputs, for example 22 to 100 Ohm, are recommended if needed. Gate drivers are assumed active-high.

Feedback must be isolated if the STM32 control ground is on the primary side. Do not directly connect secondary output ground to primary-side MCU ground unless the isolation strategy is intentional.

## Feedback Divider

Divider:

```text
R_TOP = 100 kOhm
R_BOTTOM = 10 kOhm
VADC = VOUT / 11
```

Integer formula:

```text
adc_mv  = adc_raw_filtered * 3300 / 4095
vout_mv = adc_mv * 11
```

At 24 V output, PA0 is about 2182 mV. At 28 V output, PA0 is about 2545 mV. Maximum measurable output is about 36.3 V.

ADC uses ADC1 channel 0 with DMA1_Channel1 circular buffer of 8 samples and integer IIR filtering:

```c
filtered_adc = filtered_adc + ((new_adc - filtered_adc) >> ADC_FILTER_SHIFT)
```

## PSFB Timing

TIM1 generates all power-stage timing in hardware.

- Left leg: TIM1_CH1 / TIM1_CH1N fixed 50% duty.
- Right leg: TIM1_CH2 / TIM1_CH2N fixed 50% duty.
- Power is controlled only by phase shift.
- Dead-time is generated by TIM1_BDTR.DTG.
- No GPIO bit-banging and no software dead-time.

This firmware does not use the older balanced PWM method:

```text
CCR1 = pulse_ticks
CCR2 = ARR - pulse_ticks
```

Timer:

```text
TIM1 clock = 72 MHz
FSW = 100 kHz
ARR = 72000000 / (2 * 100000) - 1 = 359
```

Phase command uses permille:

```text
phase_shift_ticks = phase_shift_permille * ARR / 1000
20% = 200 permille = about 72 ticks
```

Default phase shift is limited to 20%:

```c
#define TEST_UNLOCK_HIGH_PHASE 0
#define PHASE_SHIFT_MAX_INITIAL_PERMILLE 200U
```

Do not increase phase limit above 20% until waveforms are verified. A compile-time unlock allows 85% later:

```c
#define TEST_UNLOCK_HIGH_PHASE 1
```

## Protection

Faults clear TIM1 `MOE`, set phase shift to zero, and require reset.

Faults:

```c
FAULT_NONE
FAULT_INVALID_TARGET
FAULT_OVERVOLTAGE
FAULT_ADC_NEAR_FULL_SCALE
FAULT_FEEDBACK_LOW_OR_DISCONNECTED
FAULT_SOFTWARE_LIMIT
FAULT_EXTERNAL_BREAK
```

OVP is target x 1.25:

```text
24 V target -> 30 V OVP
28 V target -> 35 V OVP
```

ADC near full-scale fault trips above raw ADC 3900.

Output collapse handling clamps phase to a low safe value because primary current sensing is not implemented yet.

PB12 / TIM1_BKIN is reserved for future hardware current trip. Current sensing and hardware trip must be added before high-power testing.

## Bring-Up Checklist

First test with no high DC bus.

1. Use current-limited low-voltage input.
2. Do not test directly at 500 V.
3. Verify PA8/PB13 are complementary with dead-time.
4. Verify PA9/PB14 are complementary with dead-time.
5. Verify both bridge legs are fixed 50% duty.
6. Verify phase shift changes diagonal overlap.
7. Verify transformer primary positive and negative pulses are symmetrical.
8. Verify no shoot-through between high-side and low-side of each leg.
9. Verify actual MOSFET gate-source dead-time on oscilloscope.
10. Add current sensing and hardware trip before high-power testing.
